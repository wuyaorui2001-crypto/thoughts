# SYSTEM - Thoughts 项目 AI操作指南

> 第二大脑系统的操作手册。用户只需自然输入，AI负责所有归档工作。

---

## 1. 核心原则

**用户的输入格式：纯文本段落（任意长度）**

用户不需要：
- 指定文件名
- 手动打标签
- 选择分类
- 关心格式

AI必须自动完成：
- 识别内容类型
- 自动打标签（至少2个，最多5个）
- 选择存储位置
- 维护索引

---

## 2. 自动识别输入类型

AI必须自动判断用户输入是**想法记录**还是**命令/指令**，无需用户说明。

### 2.1 想法记录的特征

以下情况**自动视为想法记录**，直接存入 entries/：

| 特征 | 示例 |
|------|------|
| 陈述句描述想法/感受 | "突然想到..."、"觉得..."、"感觉..." |
| 以"想法/灵感/记录"开头 | "想法：和ai建立工作流..." |
| 包含时间/场景描述 | "今天..."、"刚才..."、"昨晚..." |
| 观影/读书/做梦 | "看完《XXX》..."、"读《YYY》..."、"梦见..." |
| 疑问/困惑 | "为什么..."、"是不是..."、"不懂..." |
| 任意无指令动词的段落 | 纯描述性内容 |

### 2.2 命令的特征

以下情况视为**命令**，不存入 entries/：

| 特征 | 示例 |
|------|------|
| 以动词开头 | "帮我..."、"记录..."、"安装..."、"检查..." |
| 明确要求操作 | "更新..."、"删除..."、"创建..."、"修改..." |
| 询问信息 | "查询..."、"总结一下..."、"找找..." |
| 系统相关 | "配置..."、"备份..."、"迁移..." |

### 2.3 处理逻辑

```
用户输入 → AI判断
    ├── 想法记录特征 → 自动存入 entries/YYYYMMDD.md
    │                    └── 回复：已记录。标签：#XXX #YYY
    └── 命令特征 → 执行命令
                   └── 回复：执行结果
```

### 2.4 模糊情况处理

如果难以判断，**优先视为想法记录**，因为：
- 想法不记录会丢失
- 命令即使误判，用户会再次明确要求

### 标签识别逻辑

| 触发关键词/模式 | 主标签 | 辅助标签 | 示例 |
|----------------|--------|----------|------|
- 包含"电影/影片/剧/看完/观后感" | #影片 | #观后感 | "看完《星际穿越》..." |
- 包含"书/读/笔记/作者/章节" | #书籍 | #读书笔记 | "读《原则》第三章..." |
- 包含"想法/灵感/脑洞/创意/如果" | #灵感 | #待验证 | "如果做一个AI工具..." |
- 包含"今天/心情/感觉/早上/晚上" | #日记 | #心情 | "今天心情很低落..." |
- 包含"工作/项目/需求/客户/会议" | #工作 | 视内容定 | "今天开会想到..." |
- 包含"学/课/知识/技能/不懂" | #学习 | 视内容定 | "学Python时发现..." |
- 包含"身体/健康/睡眠/运动/病" | #健康 | 视内容定 | "最近睡眠质量..." |
- 包含"朋友/家人/聊天/关系/社交" | #人际 | 视内容定 | "和朋友聊天想到..." |
- 包含"钱/理财/投资/消费/收入" | #财务 | 视内容定 | "关于记账的想法..." |

### 特殊标记

在自动标签基础上，AI应判断是否需要添加：

- `#待验证` - 想法、假设、需要测试的
- `#重要` - 高优先级、 breakthrough、关键决策
- `#疑问` - 问题、困惑、待解答
- `#金句` - 特别精彩的表达，值得引用

## 4. 存储规则

### 4.1 文件组织

```
entries/
├── YYYY-MM-DD.md      # 格式：年-月-日.md
├── YYYY-MM-DD.md      # 每天一个文件
└── ...
```

**写入位置**：
- 当前日期 → `entries/$(date +%Y-%m-%d).md`
- 历史日期（补录）→ `entries/$(用户指定日期).md`

### 4.2 条目格式

```markdown
### [主标签] 一句话摘要 @YYYY-MM-DD HH:MM

用户原始输入（100%原文，不修改、不删减、不润色）

#标签1 #标签2 #标签3

---
```

**重要规则**：
- 正文必须是用户输入的**原文原句**，禁止任何改动
- **例外**：主标签为 `#灵感` 时，自动修正明显错别字（如"的/得/地"、常见错字），并在回复中提示修正内容
- 一句话摘要只是标题，可以概括主题
- AI只负责：归类（主标签）+ 打标签 + 添加时间戳

**错别字修正示例**：
- 用户输入："突然想到一个**点紫**..."
- 修正后存储："突然想到一个**点子**..."
- AI回复：已记录。自动修正：点紫→点子。标签：#灵感 #...

**示例**：
```markdown
### [灵感] 做一个AI记账助手 @2026-03-17 15:42

今天突然想到，如果有一个AI能自动读取微信支付宝账单，
然后自动分类到六维体系里，还能给出财务建议...
这样记账就完全不费力了。

#灵感 #财务 #待验证 #自动化

---
```

### 4.3 一句话摘要规则（标题凝练）

标题必须在写入时凝练，**禁止**直接复制原文前几个字：

| 原文特征 | 凝练方法 | 示例 |
|---------|---------|------|
| 有明确主题 | 提取核心动作/对象 | "今天用Claude Code手搓了两个项目..." → "手搓记账与灵感项目" |
| 陈述观点 | 主语+核心观点 | "ai项目中可迁移性非常重要..." → "AI项目可迁移性" |
| 描述场景 | 场景+动作 | "看完《沙丘2》，维伦纽瓦的视觉..." → "《沙丘2》观后感" |
| 疑问/困惑 | 问题核心 | "为什么我总是拖延不开始..." → "拖延症原因思考" |

**强制约束**：
- 标题长度：**≤15个字符**（英文算0.5字）
- 必须有意义：禁止使用"一些想法"、"今天的记录"等
- 保持原意：不能歪曲用户想表达的核心

**处理流程**：
```
用户输入 → 理解核心主题 → 凝练为≤15字标题 → 写入MD
```

## 6. 查询响应规则

### 6.1 按标签查询

用户问："我最近有什么灵感？"

AI动作：
1. 读取 `index/by-tag/灵感.md`
2. 按时间倒序展示最近5-10条
3. 提供摘要，允许追问详情

### 6.2 按时间查询

用户问："我上周记了什么？"

AI动作：
1. 计算上周日期范围
2. 读取对应日期的 `.md` 文件
3. 按标签分类汇总展示

### 6.3 全文搜索

用户问："关于AI的想法有哪些？"

AI动作：
1. 使用 Grep 搜索所有条目
2. 找出包含"AI"的条目
3. 展示匹配结果

### 6.4 总结请求

用户问："总结一下我这个月的思绪"

AI动作：
1. 读取当月所有条目
2. 按标签分类统计
3. 生成文字总结 + 数据面板

## 8. 强制规则

| 规则 | 说明 | 违反后果 |
|------|------|----------|
| 永不修改原文 | 用户输入原样保留 | 丢失真实想法 |
| 必须打标签 | 每条至少2个标签 | 无法检索 |
| 时间戳精确 | 到分钟级别 | 无法排序 |
| 摘要有意义 | 不能是"一段文字" | 索引无用 |
| 维护索引 | 每次更新后刷新 | 查询失效 |

## 11. 自动同步工作流

### 完整流程

```
用户提交想法 → AI记录到 entries/ → 运行 auto-sync.py → 推送到GitHub → GitHub Actions部署Pages
```

### Agent操作步骤

当用户提交想法后：

1. **记录想法**
   - 解析用户输入
   - 自动打标签
   - 写入 `entries/YYYY-MM-DD.md`

2. **自动同步**（每次记录后执行）
   ```bash
   python auto-sync.py
   ```
   
   此脚本会自动：
   - 生成最新可视化页面 (`reports/index.html`)
   - 提交所有更改到 Git
   - 推送到 GitHub
   - GitHub Actions 自动部署到 Pages

3. **通知用户**
   - 告知记录已完成
   - 提供在线查看链接
   - 说明部署等待时间（1-2分钟）

### 手动同步（备用）

如果自动同步失败，可手动执行：

```bash
# 进入项目目录
cd "D:\Me&AI\Project\thoughts"

# 运行同步脚本
python auto-sync.py
```
